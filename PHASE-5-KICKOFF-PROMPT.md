What Phase 5 is: the authoring UI that lets people BUILD and EDIT the forms that the rest of the platform already renders and stores as evidence. The forms engine already exists (Phase 2) and must be reused, not rebuilt:

Reuse (do NOT rebuild): the schema types (lib/form-schema.ts) and the shape { schemaVersion, sections: [ { id, title, description?, fields: [ { key, type, label, required?, help?, placeholder?, options?, validation?, visibleWhen? } ] } ] }; the shared renderer (components/forms/form-renderer.tsx); the validator (lib/form-validate.ts, cleanAnswers/validateAnswers); the formatter (lib/form-format.ts); the evidence pipeline (immutable evidence, pinned form_version_id + embedded schema_snapshot, on-demand branded PDF). The builder AUTHORS the exact same schema these already consume — a live preview should just mount the existing FormRenderer.

Field types the builder must support (the full v1 set the renderer already handles): short text, long text, number, date, single select, multi select, radio, checkbox, section heading, signature, file upload, plus conditional logic (visibleWhen). Required fields, help text, placeholders, options, and per-field validation are all part of the schema already.

Versioning (hard requirement, already modelled): forms have immutable form_versions (form_id, version, schema, status draft|published) and forms.current_version. Evidence pins the form_version_id AND embeds a schema_snapshot, so past evidence renders identically forever. The builder must respect this: editing a published form creates a NEW draft version; publishing makes it the current version; existing evidence is never altered. Never mutate a published version's schema in place.
